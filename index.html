บทที่ 3

ตัวดำเนินการคำนวณและเปรียบเทียบ

การเคลื่อนย้ายสสารและการจำลองใน Simulated Universe ต้องใช้ความสามารถในการคำนวณที่รวดเร็ว ตัวดำเนินการ (Operators) คือกลไกเชื่อมโยงตัวเลขเข้าด้วยกัน

1. ตัวดำเนินการคำนวณ (Arithmetic Operators)

สัญลักษณ์คำนวณทางคณิตศาสตร์ใน Python ที่ช่วยให้เราสั่งคอมพิวเตอร์ประมวลผลเลขยากๆ ได้:

  • +, - : บวก และ ลบ
  • * : คูณ (ห้ามใช้ x เด็ดขาด!)
  • / : หาร (ผลลัพธ์เป็น float เสมอ)
  • // : หารเอาส่วน/หารปัดเศษลง (ปัดเศษทศนิยมทิ้ง)
  • % : หารเอาเศษ (Modulo - มีประโยชน์มากสำหรับการหาเลขคู่/คี่)
  • ** : ยกกำลัง (เช่น 2**3 จะมีค่าเท่ากับ 8)

2. ตัวดำเนินการเปรียบเทียบ (Comparison Operators)

ใช้เพื่อเปรียบเทียบค่าสองฝั่ง และจะส่งค่าผลลัพธ์กลับมาเป็นตรรกะ True หรือ False เท่านั้น:

  • == : เท่ากับ (เปรียบเทียบ)
  • != : ไม่เท่ากับ
  • >, < : มากกว่า, น้อยกว่า
  • >=, <= : มากกว่าหรือเท่ากับ, น้อยกว่าหรือเท่ากับ

ข้อควรระวังอันตรายถึงชีวิตหุ่นเชิด!

จำไว้ให้ขึ้นใจ! เครื่องหมาย = ตัวเดียวหมายถึง "การกำหนดค่า" (Assign Value) ใส่กล่องตัวแปร ส่วนเครื่องหมาย == สองตัวหมายถึง "การเปรียบเทียบค่าว่าเท่ากันหรือไม่" (Comparison)

# กำหนดค่า
mana = 100 

# เปรียบเทียบค่า (จะได้ผลลัพธ์เป็น True)
is_full = (mana == 100)

บทที่ 4

ประตูแยกแห่ง Aeon (If-Elif-Else)

การตัดสินใจคือแก่นแท้ของปัญญา ประตูแยกแห่ง Aeon ควบคุมทิศทางไหลของกระแสพลังตามความสมบูรณ์และเงื่อนไขที่เราสลักไว้

1. โครงสร้างควบคุมการตัดสินใจ (Conditionals)

Python ใช้ชุดควบคุม if (ถ้า), elif (หรือถ้า - ย่อมาจาก else if), และ else (นอกเหนือจากนั้น) เพื่อให้โปรแกรมทำงานตามเงื่อนไขที่ต่างกัน

score = 75

if score >= 80:
    print("คุณได้รับพรจาก Aeon Aha (Elation)")
elif score >= 50:
    print("คุณได้รับพรจาก Aeon Lan (Hunt)")
else:
    print("คุณถูกทำลายล้างโดย Aeon Nanook (Destruction)")

2. กฎการย่อหน้า (Indentation) ในภาษา Python

แตกต่างจากภาษาอื่นที่ใช้ปีกกา {} ไวยากรณ์ Python จะใช้ "การย่อหน้า" (Indentation - แนะนำให้กดปุ่ม Tab 1 ครั้ง) ในการบ่งบอกขอบเขตว่าโค้ดบรรทัดใดอยู่ภายใต้เงื่อนไขนั้นๆ

3. ตัวเชื่อมตรรกะจำลอง (Logical Operators)

ในกรณีที่มีหลายเงื่อนไขรวมกัน เราสามารถใช้คำเหล่านี้เข้ามาเชื่อม:

  • and: เงื่อนไขต้องเป็นจริง ทั้งคู่ จึงจะผ่าน (เช่น mana > 50 and hp > 0)
  • or: ขอแค่เงื่อนไขใดเงื่อนไขหนึ่งเป็นจริง อย่างน้อยหนึ่งข้อ ก็ผ่านได้
  • not: พลิกกลับค่าตรรกะ (True เป็น False, False เป็น True)

บทที่ 5

วงแหวนวนซ้ำ (For & While Loops)

การวิจัยทางเวทมนตร์ต้องอาศัยการทดลองซ้ำนับพันๆ ครั้ง วงแหวนวนซ้ำ (Loops) ช่วยให้เราเขียนคำสั่งสั่งงานซ้ำๆ ได้อย่างง่ายดายด้วยสลักอักขระเพียงไม่กี่บรรทัด

1. วงแหวนมีรอบกำหนด: For Loop & range()

For Loop เหมาะสำหรับการทำงานซ้ำที่มีจำนวนครั้งที่ระบุชัดเจน มักทำงานร่วมกับฟังก์ชัน range(start, stop) (รอบการทำจะหยุดก่อนถึงตัวเลข stop เสมอ)

# วนรอบรดน้ำต้นไม้วิเศษ 5 รอบ (i จะมีค่าตั้งแต่ 1 ถึง 5)
for i in range(1, 6):
    print("รดน้ำวิเศษรอบที่", i)

2. วงแหวนเงื่อนไขขอบเขต: While Loop

While Loop จะวนซ้ำการทำคำสั่งไปเรื่อยๆ ตราบใดที่เงื่อนไขระบุหลังคำว่า while ยังคงประเมินเป็นจริง (True)

คำเตือนภัยพิบัติ: อย่าลืมสร้างคำสั่งเพิ่มค่าตัวนับรอบการทำงาน (เช่น count += 1) ในตอนท้ายบล็อกวงลูป มิฉะนั้นโปรแกรมจะติดอยู่ใน **ลูปไม่สิ้นสุด (Infinite Loop)** ส่งผลให้ห้องแล็บระบบค้างและระเบิดได้! 
count = 1
while count <= 3:
    print("ทดลองหุ่นเชิดร่างที่", count)
    count += 1 # ตัวนับพลังงานเพื่อให้เงื่อนไขกลายเป็น False ในที่สุด

  • Return Value (ส่งคืนผลลัพธ์): คีย์เวิร์ด return ส่งค่าสรุปสุดท้ายหลุดออกไปหาจุดที่เรียกใช้
def calculate_spell_power(mana, ratio):

# เรียกใช้งานฟังก์ชันและนำผลมาบันทึก

1. แนวคิด Object-Oriented Programming (OOP)

OOP คือแนวทางการเขียนโปรแกรมที่จำลองเอาองค์ประกอบในชีวิตจริงหรือห้องทดลองมาอยู่ในรูปแบบของวัตถุที่มีคุณลักษณะและการทำงานในตัวเอง

  • Class (พิมพ์เขียว): เสมือนแบบวาดพิมพ์เขียวโครงกระดูกเหล็กในกระดาษวิศวกรรม
  • Object (หุ่นเชิดของจริง): ตัวหุ่นเชิดที่ประกอบขึ้นมาสำเร็จรูปตามขนาดและแบบแปลนนั้นและมีตัวตนจริงในคอมพิวเตอร์

2. การใช้เมธอดสร้างเริ่มต้น __init__ และคีย์เวิร์ด `self`

  • __init__ : คอนสตรัคเตอร์ (Constructor) เมธอดที่จะทำงานทันทีที่ตัววัตถุ (Object) ถูกสร้างขึ้น มีหน้าที่ตั้งค่าสถิติเริ่มต้นของชิ้นงาน
  • self : ตัวแทนที่อ้างอิงถึงตัวแปรวัตถุชิ้นนั้นๆ เอง ช่วยให้แต่ละหุ่นเก็บค่าระดับพลังและชื่อแยกของตนเองได้อย่างอิสระ
class Puppet:
    def __init__(self, name, score):
        self.name = name       # คุณสมบัติ (Attribute)
        self.score = score     # คุณสมบัติ
        
    def kurukuru(self):        # ความสามารถ (Method)
        return f"{self.name} กำลังหมุนตัว คุรุคุรุ~! ทักษะปัญญา: {self.score}"

# นำพิมพ์เขียวมาปั้นตัวหุ่นเชิดจริง (Objects)
puppet_a = Puppet("หุ่นไม้เบอร์ 1", 85)
puppet_b = Puppet("หุ่นแก้วเบอร์ 2", 99)

print(puppet_a.kurukuru()) # หุ่นไม้เบอร์ 1 กำลังหมุนตัว คุรุคุรุ~! ทักษะปัญญา: 85

การวิจัยที่ดีต้องต่อยอดได้ การสืบทอด (Inheritance) ช่วยให้เราสร้างพิมพ์เขียวลูกแบบใหม่ที่มีความสามารถพิเศษเพิ่มขึ้นโดยดึงคุณสมบัติพื้นฐานเดิมของคลาสแม่มาทั้งหมด!

1. การสืบทอดความสามารถของคลาสลูก (Inheritance)

เราส่งค่าชื่อคลาสแม่เข้าไปในวงเล็บหลังการประกาศคลาสลูก เพื่อบอกความสัมพันธ์การสืบทอด

# คลาสแม่ดั้งเดิม
class BasePuppet:
    def __init__(self, name):
        self.name = name
    def stand_by(self):
        return f"{self.name} ประจำสถานีทดสอบ"

# คลาสลูกที่เพิ่มเติมความสามารถสู้รบ
class CombatPuppet(BasePuppet):
    def fire_laser(self):
        return f"{self.name} ยิงลำแสงสีม่วงพาสเทลอัดศัตรู!"

2. การใช้ฟังก์ชัน `super().__init__()`

ในการประกอบโครงร่างคลาสลูก บางครั้งเรายังต้องรันกระบวนการจัดตั้งเบื้องต้นของคลาสแม่เสียก่อน วิธีการคือเรียกใช้ฟังก์ชัน super() เพื่อส่งพลังไปยังคอนสตรัคเตอร์คลาสหลัก

class CombatPuppet(BasePuppet):
    def __init__(self, name, laser_power):
        super().__init__(name) # อัญเชิญสร้างจากคลาสแม่
        self.laser_power = laser_power # สถิติตัวเสริมของคลาสลูกเอง

คุณลักษณะ: name

ความสามารถ: stand_by()

ได้รับสืบทอด
Class: CombatPuppet (คลาสลูก)

คุณลักษณะ: name, laser_power เพิ่ม

ดักจับสิ่งปนเปื้อน (Exceptions)

ไม่มีสิ่งประดิษฐ์ใดไร้ตำหนิ ในสภาวะแวดล้อมจริง ข้อผิดพลาด (Exceptions) เช่น การพิมพ์ตัวอักษรแทนตัวเลข การหารด้วยศูนย์ ย่อมเกิดขึ้นได้และอาจทำให้ระบบจักรวาลจำลองล่มลง!

1. โครงสร้างป้องภัยสะกดพัง: Try-Except

เราห่อหุ้มคำสั่งที่มีความเสี่ยงว่าจะแตกพังไว้ในตึกคำสั่ง try และระบุคำแนะนำสำรองไว้ในตึก except หากพังจริงระบบจะไม่เด้งค้าง แต่จะเลี่ยงไฟมาทำบล็อกสำรองทันที

try:
    number = int(input("ป้อนตัวเลขปัญญา: "))
    result = 100 / number
    print("ผลลัพธ์คำนวณ:", result)
except ZeroDivisionError:
    print("ไม่สามารถเอาเลข 0 มาหารระบบพลังได้!")
except ValueError:
    print("ขออภัย! คาถาต้องการเลขจำนวนจริง ไม่รับตัวอักษร")

2. คันดักอัปเกรด: Else & Finally

  • else : ทำงานเฉพาะกรณีที่คำสั่งในบล็อก try รันได้อย่างราบรื่นและไม่มีสิ่งแปลกปลอมพังเลย
  • finally : บล็อกรับประกันภัยที่จะ **รันเสมอแน่นอนร้อยเปอร์เซ็นต์** ไม่ว่าโค้ดจะรันผ่านฉลุยหรือเกิดการแตกพังบานปลายในจุดทดลองก็ตาม มักใช้เปิดปิดระบบควบคุมทรัพยากร

เครื่องจำลองเกราะป้องกันโปรแกรมล่ม (Shield Simulator)

ทดลองป้อนตัวเลข "ตัวหาร (Divisor)" และเลือกเปิด/ปิดเกราะสะกด Try-Except สังเกตปฏิกิริยาของระบบ:

เกราะ Try-Except:

[System] พร้อมตรวจหาจุดปนเปื้อน...

บทที่ 12

การเปิดขอบเขตโลกภายนอก (Files & Modules)

บทเรียนสุดท้ายของโปรโตคอลการจำลอง! หุ่นเชิดอัจฉริยะต้องไม่ถูกจองจำอยู่เพียงในหน่วยความจำชั่วคราว เราต้องสามารถจดข้อมูลลงม้วนบันทึกถาวร และอัญเชิญโมดูลภายนอกมาใช้ได้

1. การเขียนและอ่านม้วนเอกสารไฟล์: with open()

เราเปิดและปิดประตูไฟล์โดยอัตโนมัติด้วยโครงสร้าง with open() เพื่อรับประกันว่าไฟล์จะถูกเก็บกวาดเข้าคลังหลังใช้งานเสร็จสิ้น

# เขียนสารอักขระลงสมุดบันทึกวิเศษ
with open("lab_log.txt", "w", encoding="utf-8") as file:
    file.write("การทดสอบหุ่นเชิดอัจฉริยะ บทที่ 12: สำเร็จราบรื่น!")

# เปิดอ่านอภิมหาคลังบันทึกที่บันทึกไว้
with open("lab_log.txt", "r", encoding="utf-8") as file:
    content = file.read()
    print("คลังบันทึกจดบันทึกไว้ว่า:", content)

2. การนำเข้าโมดูลภายนอกที่เป็น Standard Libraries

ดึงเอาวิทยาศาตร์ระดับโลกที่มีอยู่แล้วมาประกบสเกลใช้งานผ่านคำสั่ง import:

  • import random : เปิดการสุ่มคาถา สุ่มไอเทม
  • import time : ใช้หยุดเวลาเพื่อพักร่างหุ่นเชิด (เช่น time.sleep(1))

3. การติดตั้งคลังแสงผ่าน Terminal ด้วยเครื่องมือ pip

ในโลกเวทมนตร์ Python มีสมาคมมักเขียนคาถาสำเร็จรูปแจกจ่ายให้เราใช้งานได้ฟรี เราเปิดแอป Terminal ในเครื่องแล้วสลักคาถาสั่งดาวน์โหลดได้เลย:

> pip install requests numpy pandas

ตัวอย่าง: requests ช่วยเรายิงจดหมาย API เชื่อมต่อกับเว็บภายนอกทั่วทั้งดาวเคราะห์โลก!
