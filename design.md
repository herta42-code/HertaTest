## 1. Visual & Color Palette Design (อ้างอิงจากสไลด์)
โทนสีหลักของเว็บไซต์ได้รับแรงบันดาลใจจากธีมห้องปฏิบัติการเวทมนตร์ของ Herta (สไตล์อนิเมะพาสเทล ม่วง-ชมพู-ครีม ให้ความรู้สึกน่ารัก อบอุ่น แต่แฝงไปด้วยความลึกลับและปัญญา)

*   **Primary Background (โทนพื้นหลังหลัก):** `#FDFBF7` (สีครีมสว่าง/กระดาษเก่า ช่วยให้อ่านเนื้อหาโค้ดได้สบายตา ไม่ล้าสมัย)
*   **Primary Brand Color (สีหลักประจำแบรนด์):** `#8E7AB5` / `#7B66FF` (สีม่วงลาเวนเดอร์เข้มและม่วงเวทมนตร์ สำหรับปุ่มหลัก, หัวข้อใหญ่ และ UI Highlights)
*   **Secondary Accent (สีรอง/เน้นย้ำ):** `#E198B7` / `#D4ADFC` (สีชมพูพาสเทลและม่วงอ่อน สำหรับองค์ประกอบคำแนะนำตัวละคร Herta Callout)
*   **Text & Typography (สีตัวอักษร):** 
    *   Heading/Title: `#332C3F` (สีม่วงเข้มเกือบดำ เพื่อความเปรียบต่างสูง อ่านง่าย)
    *   Body Text: `#4F4A57` (เทาเข้มอมม่วง)
*   **Code Block Environment:** Background `#2D2736` (สีดาร์กเพอเพิล) พร้อม Syntax Highlighting โทนสีนีออนพาสเทล (ชมพู ฟ้า เขียวตองอ่อน)

## 2. Website Structure & Navigation Map (โครงสร้างสารบัญเว็บ)

เว็บไซต์จะถูกออกแบบเป็น **Single-Page Application (SPA) พร้อม Sidebar Navigation ด้านซ้าย** เพื่อให้ผู้เรียนสามารถคลิกดูแต่ละบทเรียนได้อย่างลื่นไหลโดยไม่ต้องโหลดหน้าใหม่

```
[ ลิงก์ส่วนหัว: โลโก้สมาคมอัจฉริยะ #83 | หน้าแรก | คู่มือจำลอง | ติดต่อสมาคม ]
-------------------------------------------------------------------------
(Sidebar ซ้าย: สารบัญ 12 บท)         | (Main Content Area: ส่วนแสดงเนื้อหาหลัก)
- บทนำ: แผนที่การเดินทางท่องจักรวาลจำลอง  | 
- บทที่ 1: ติดตั้งห้องแล็บจำลอง (Setup)  | [กล่องข้อความทักทายจาก Herta (ต้อนรับ)]
- บทที่ 2: เวทมนตร์พื้นฐาน & ตัวแปร        | 
- บทที่ 3: ตัวดำเนินการทางคณิตศาสตร์      | [เนื้อหาตามบทเรียนที่เลือกจาก Sidebar]
- บทที่ 4: ประตูทางเลือก (Conditionals)  | 
- บทที่ 5: วงแหวนวนซ้ำ (Loops)         | [Syntax ไฮไลต์โค้ด + กล่องคำแนะนำคำเตือน]
- บทที่ 6: กล่องเก็บของอาเรย์ (Lists)     | 
- บทที่ 7: ดัชนีสารานุกรม (Dictionaries) | [กล่องสรุปข้อคิดเห็น "HERTA Says!"]
- บทที่ 8: มนตราเรียกใช้ซ้ำ (Functions)    |
- บทที่ 9: พิมพ์เขียวหุ่นเชิด (OOP Basics) | 
- บทที่ 10: การสืบทอดพิมพ์เขียว (OOP Class)| 
- บทที่ 11: ดักจับสิ่งปนเปื้อน (Exceptions) | 
- บทที่ 12: การเปิดขอบเขตโลกภายนอก       | 
- บทสรุปการทดสอบ: ความสำเร็จขั้นแรก       | 
```

### 3.2 Core Lessons Layout (พาดหัวหลักและหัวข้อย่อยทั้ง 12 บท)

#### **บทที่ 1: การติดตั้งห้องแล็บจำลอง (Setup)**
*   **หัวข้อย่อย:**
    1. การติดตั้งตัวแปลภาษา Python (`python.org` + ติ๊กถูก *Add Python to PATH*)
    2. โต๊ะเขียนมนตรา (แนะนำ Visual Studio Code)
    3. การติดตั้งส่วนเสริม Python (Microsoft Extensions)
*   **Interactive Feature:** ปุ่มสลับระบบปฏิบัติการ (Windows / macOS) เพื่อแสดงวิธีกดตรวจสอบผ่าน Terminal ด้วยคำสั่ง `python --version`
*   **Herta's Note:** *"ถ้าเจอบั๊กระหว่างติดตั้ง ลองพิมพ์หาในเน็ตดู... แต่ถ้า PATH ผิดก็แค่ลบลงใหม่ อย่าเพิ่งถอดใจล่ะ!"*

#### **บทที่ 2: เวทมนตร์พื้นฐาน & ตัวแปร**
*   **หัวข้อย่อย:**
    1. การสร้างกล่องข้อมูล (Variables) และเครื่องหมาย `=`
    2. ประเภทข้อมูลหลัก (`str`, `int`, `float`, `bool`)
    3. การแสดงผล (`print()`) และการรับค่า (`input()`)
*   **Component:** ตารางเปรียบเทียบประเภทตัวแปรพร้อมโค้ดตัวอย่างในภาษา Python และ **กฎเหล็กการตั้งชื่อตัวแปร** (ห้ามขึ้นต้นด้วยตัวเลข, ห้ามมีช่องว่าง, หลีกเลี่ยงภาษาไทย)

#### **บทที่ 3: ตัวดำเนินการคำนวณและเปรียบเทียบ**
*   **หัวข้อย่อย:**
    1. ตัวดำเนินการคำนวณ (`+`, `-`, `*`, `/`, `//`, `%`, `**`)
    2. ตัวดำเนินการเปรียบเทียบ (`==`, `!=`, `<`, `>`, `<=`, `>=`)
*   **Warning Callout:** ข้อควรระวังระหว่าง `=` (กำหนดค่า) และ `==` (เปรียบเทียบค่า)

#### **บทที่ 4: ประตูแยกแห่ง Aeon (If-Elif-Else)**
*   **หัวข้อย่อย:**
    1. การควบคุมการตัดสินใจผ่านโครงสร้าง `if`, `elif`, `else`
    2. กฎการย่อหน้า (Indentation) ในภาษา Python
    3. ตัวเชื่อมตรรกะจำลอง (`and`, `or`, `not`)
*   **Visual Element:** แผนผังการตัดสินใจ (Flowchart) จำลองคะแนนการรับพรจาก Aeon

#### **บทที่ 5: วงแหวนวนซ้ำ (For & While Loops)**
*   **หัวข้อย่อย:**
    1. โครงสร้าง `For Loop` ร่วมกับฟังก์ชัน `range()`
    2. โครงสร้าง `While Loop` และการเพิ่มค่าตัวนับป้องกันลูปไม่สิ้นสุด (Infinite Loop)
    3. คำสั่งควบคุมพิเศษ: `break` และ `continue`

#### **บทที่ 6: กล่องสัมภาระเดินทาง (Lists)**
*   **หัวข้อย่อย:**
    1. การจัดเก็บข้อมูลเป็นอาเรย์เรียงลำดับด้วยดัชนี (Index เริ่มจาก 0)
    2. การเข้าถึงข้อมูลจากท้ายสุดด้วยค่าลบ (เช่น `[-1]`)
    3. การอัปเดตและขยายขนาดลิสต์ด้วยฟังก์ชัน `.append()` และการใช้ `len()`
    4. ฟังก์ชันสไลซ์ข้อมูล (List Slicing `[start:end]`)

#### **บทที่ 7: สารานุกรมวิจัย (Dictionaries)**
*   **หัวข้อย่อย:**
    1. การเก็บข้อมูลรูปแบบคู่คู่คีย์และข้อมูล `{Key: Value}`
    2. คำสั่งช่วยเหลือที่พบบ่อย (`.keys()`, `.values()`, `.get()`)
*   **Herta's Alert:** คีย์ในดิกชันนารีห้ามซ้ำกันเด็ดขาด หากใส่ซ้ำค่าเก่าจะถูกเขียนทับทันที!

#### **บทที่ 8: จารึกคาถาเพื่อเรียกใช้ซ้ำ (Functions)**
*   **หัวข้อย่อย:**
    1. การประกาศฟังก์ชันด้วยคีย์เวิร์ด `def`
    2. การรับวัตถุดิบ (Parameters) และการส่งค่าคืนกลับ (`return`)
    3. ความเข้าใจเรื่องขอบเขตตัวแปร (Local Variable vs Global Variable)

#### **บทที่ 9 (ขั้นกลาง): แบบแปลนสร้างหุ่นเชิด (OOP Basics)**
*   **หัวข้อย่อย:**
    1. แนวคิด Object-Oriented Programming (OOP)
    2. ความแตกต่างระหว่าง `Class` (พิมพ์เขียว) และ `Object` (หุ่นเชิดจำลอง)
    3. การใช้เมธอดสร้างคุณสมบัติแรกเริ่ม `__init__` การใช้ `self` และแอดทริบิวต์ (Attribute)

#### **บทที่ 10 (ขั้นกลาง): การสืบทอดร่างหุ่นเชิด (Inheritance)**
*   **หัวข้อย่อย:**
    1. การสืบทอดความสามารถจากคลาสแม่ (Parent class) สู่คลาสลูก (Child class)
    2. การใช้ฟังก์ชัน `super().__init__()` เพื่อดึงการตั้งค่าดั้งเดิม
    3. การเพิ่มเมธอดเฉพาะตัวเพื่อขยายขีดความสามารถของคลาสลูก

#### **บทที่ 11: เกราะกำบังข้อยกเว้น (Try-Except)**
*   **หัวข้อย่อย:**
    1. โครงสร้างการครอบดักข้อผิดพลาดป้องกันโปรแกรมล่มด้วย `try-except`
    2. การจัดการข้อยกเว้นเฉพาะเจาะจง (`ZeroDivisionError`, `ValueError`)
    3. การใช้งานบล็อกเสริม `else` และ `finally` (ทำงานเสมอไม่ว่าจะพังหรือไม่)

#### **บทที่ 12 (ขั้นกลาง): การเชื่อมต่อโลกภายนอก (Files & Modules)**
*   **หัวข้อย่อย:**
    1. การอ่านและเขียนไฟล์พื้นฐานด้วยชุดคำสั่ง `with open() as file:`
    2. การนำเข้าโมดูลพลังงานภายนอกที่เป็น Standard Libraries (`import random`, `import time`)
    3. การติดตั้งคลังแสงเสริมภายนอกผ่าน Terminal ด้วยเครื่องมือ `pip` (เช่น `requests`, `numpy`, `pandas`)

## 4. Key UI Components & Interactions (ลูกเล่นระบบเว็บที่ออกแบบ)

1.  **Herta Interactive Widget:** ที่มุมขวาของทุกบทเรียนจะมีไอคอนหรือแอนิเมชันรูป Herta คอยหมุนตัวแบบ "Kurukuru" เมื่อผู้เรียนสามารถกดทำแบบทดสอบท้ายบทผ่าน
2.  **Run Code Sandbox:** มีหน้าต่างเล็ก ๆ ให้ทดลองพิมพ์โค้ด Python พื้นฐานตามตัวอย่างในสไลด์และประมวลผลได้ทันทีบนหน้าเว็บ (โดยเฉพาะส่วนของ `print`, `if-else` และ `loops`)
3.  **Progress Tracking Bar:** แถบสีม่วงพาสเทลด้านบนสุดของหน้าต่างสารบัญ เพื่อแสดงว่าผู้เรียนผ่านการทดสอบจำลองมาแล้วกี่เปอร์เซ็นต์ (จากทั้งหมด 12 บทเรียน)
